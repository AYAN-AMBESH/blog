## Baseline and Error Checks

Test each user-controllable parameter one by one.

Start with a quote-break test:

```text
'
```

If behavior changes, errors appear, or response rendering breaks, you may have
an injectable context.

Look for DB-related messages in HTTP responses, logs, or debug traces.

---

## Boolean-Based Test

Run controlled comparisons in the same parameter:

```text
1=1
1=2
```

If the application response differs between true and false conditions without
throwing visible SQL errors, this can indicate Boolean-based Blind SQLi.

---

## Timing-Based Follow-up

If true/false responses look the same, test with time delays:

```text
SLEEP(5)
pg_sleep(5)
WAITFOR DELAY '0:0:5'
```

A consistent delay pattern suggests Blind SQL Injection.

---

## Identify Database Type

Use error formats and supported syntax/functions to fingerprint the DB engine.
